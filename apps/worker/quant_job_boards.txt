FEASIBILITY FINDINGS (researched 2026-06-10, verified against live endpoints)
================================================================
BUILT this session (adapters + tests, verified live): workday, pinpoint  (REUSABLE vendor APIs).
  -> added to config: Arrowstreet, Millennium (workday); Wolverine (pinpoint).
DECISION: build only reusable multi-firm ATS adapters; SKIP single-firm self-hosted scrapers
  (Jane Street, DE Shaw) — low reuse + brittle (DE Shaw scrapes Next.js HTML). Feasible if ever needed.
Still open below: AQR/Peak6 (not Workday), Jane Street + DE Shaw (single-firm, deferred by choice),
  G-Research+Rentec (JSON-LD), Citadel (Playwright), SimplifyJobs discovery, other unsupported ATS.

WORKDAY adapter — feasible, ~50 lines, public CXS API (POST /wday/cxs/{tenant}/{site}/jobs),
  no auth/anti-bot. Slug = "tenant/dc/site". CAVEAT: description needs an N+1 detail call per
  posting. Of the 4 Workday firms only ARROWSTREET is live (arrowstreetcapital/wd5/Campus_Careers,
  3 roles); MILLENNIUM (mlp/wd5/mlpcareers) is a valid board but empty now. AQR is NOT Workday
  (custom careers.aqr.com/api/jobs, ~44 roles — needs its own small adapter). PEAK6 is NOT Workday
  (Ongig, token-gated — hard).

PINPOINT adapter — Wolverine uses Pinpoint, not Lever: GET https://wolve.pinpointhq.com/postings.json
  (clean JSON, 14 roles incl. quant/C++). Trivial adapter, mirrors greenhouse/lever shape.

SELF-HOSTED (Group B) — NOT one general scraper. It's per-site enumerators + 1 shared schema.org
  JSON-LD detail extractor. Tiers:
   - Easy, clean JSON, no new deps:  Jane Street (https://www.janestreet.com/jobs/main.json, 209 roles);
     DE Shaw (__NEXT_DATA__ in page HTML, 73 roles).
   - Medium, needs an HTML parser (selectolax/bs4) + shared JSON-LD extractor:  G-Research (61);
     Renaissance/Rentec (8).
   - Hard, needs Playwright (Cloudflare + JS-rendered):  Citadel Securities (59). Defer.
   - Skip:  Headlands (email only, recruiter@headlandstech.com).

GITHUB AGGREGATORS:
   - SimplifyJobs/New-Grad-Positions — rich JSON feed (raw .../dev/.github/scripts/listings.json,
     ~2,001 active+visible; "Quant" category ~129 active). Best as DISCOVERY mode: extract ATS+slug
     from each row's url (~26% route to our greenhouse/lever/ashby adapters) and fetch real
     descriptions via existing adapters. Direct-ingest is NOT viable (feed has no job description ->
     scorer/screen break). LICENSE CAVEAT: repo is unlicensed (default copyright), commercial product;
     discovery mode is the cleanest posture (descriptions re-fetched from each ATS's own public API).
   - jobright-ai/2026-Software-Engineer-New-Grad — README-only, links to gated jobright.ai pages
     (no ATS exposed, no descriptions). NOT feasible. Skip.
